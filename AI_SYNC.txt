CRITICAL SECURITY VULNERABILITIES
================================================================================

1. CRITICAL: FormulaEngine.evaluate() uses eval() with __builtins__ override
   File: oracle/symbolic/modifiable.py:232
   Issue: `result = eval(formula, {"__builtins__": {}}, safe_vars)` - This is
   exploitable. Even with __builtins__ override, sophisticated attacks can
   bypass this. The safe_eval module exists at oracle/utils/safe_eval.py but
   is NOT used by FormulaEngine.
   Directive: REPLACE eval() in FormulaEngine.evaluate() with calls to
   safe_eval.safe_eval() from oracle/utils/safe_eval.py. The FormulaEngine
   must NEVER use raw eval(). This is a hard requirement.

2. CRITICAL: html.escape() applied to user questions destroys non-ASCII content
   File: oracle/sanitizer.py:22
   Issue: `question = html.escape(question)` destroys Arabic, Hebrew, Persian,
   and Chinese characters which are the PRIMARY input of this oracle system.
   html.escape converts non-ASCII entities, breaking gematria calculations,
   character analysis, and all symbolic processing.
   Directive: REMOVE html.escape() from sanitize_question(). The sanitizer
   should only: (a) enforce MAX_QUESTION_LENGTH, (b) strip leading/trailing
   whitespace, (c) log suspicious patterns. Do NOT escape Unicode characters.

3. CRITICAL: SQLite database connections are never properly managed
   Files: oracle/memory/archive.py, oracle/memory/chromosome_archive.py,
   oracle/memory/experiment_ledger.py, oracle/memory/mutation_bank.py
   Issue: Every method opens a new sqlite3.connect() and commits individually.
   No connection pooling, no WAL mode, no proper transaction management.
   Concurrent access will cause database locking.
   Directive: Implement a singleton database manager class with:
   - Connection pooling or WAL journal mode
   - Context manager pattern (with statement) for all DB operations
   - Proper exception handling with rollback
   - Thread-safe connection handling
   Use `PRAGMA journal_mode=WAL` and `PRAGMA busy_timeout=5000`

4. CRITICAL: md5 used for content hashing in multiple files
   Files: oracle/runtime/executor.py:172, oracle/evolution/rule_invention.py,
   oracle/evolution/mutation.py:326
   Issue: MD5 is cryptographically broken. For a system that generates
   "oracle outputs" that may be referenced, this is unacceptable.
   Directive: Replace ALL md5 usage with hashlib.sha256(). Search all files
   for `hashlib.md5` and replace with `hashlib.sha256`.

ERROR HANDLING AND RESILIENCE
================================================================================

5. HIGH: Bare except clauses silently swallow all errors
   Files: oracle/symbolic/registry.py:124 (`except Exception: pass`),
   oracle/evaluation/fitness.py (multiple `except Exception` blocks),
   oracle/evolution/engines/shared.py:55
   Issue: Silent error swallowing makes debugging impossible. In production,
   errors are INFO, not silence.
   Directive: Replace ALL bare `except Exception: pass` with:
   ```python
   except ImportError as e:
       logger.debug("Optional dependency not available: %s", e)
   except Exception as e:
       logger.warning("Module %s failed to load: %s", mod_path, e)
   ```
   NEVER use `pass` in except blocks. ALWAYS log the exception.

6. HIGH: OraclePipeline.__init__ silently creates data directory with os.makedirs
   File: oracle/runtime/executor.py:78
   Issue: `os.makedirs(os.path.dirname(...), exist_ok=True)` runs inside
   __init__ which can fail on read-only filesystems or permission issues.
   Directive: Move directory creation to an explicit `setup()` method or
   use a lazy initialization pattern. Log the directory creation. Handle
   OSError properly.

7. HIGH: No timeout protection on chromosome execution
   File: oracle/genome/chromosome.py:78 (execute method)
   Issue: chromosome.execute() can hang indefinitely if a symbolic system
   enters an infinite loop. The ExecutionSandbox exists but is NOT used
   by the actual execution path.
   Directive: Integrate ExecutionSandbox into the execution pipeline.
   All chromosome.execute() calls MUST go through the sandbox with a
   configurable timeout. The sandbox must kill long-running computations.

8. HIGH: _next_prime() in modifiable.py can infinite loop
   File: oracle/symbolic/modifiable.py:276-278
   Issue: `while True: if all(n % i != 0 ...): return n; n += 1` can
   theoretically run forever for very large n values.
   Directive: Add a maximum iteration limit (e.g., 10000 iterations).
   Return the best candidate found if limit is reached.

9. HIGH: FormulaEngine.random_formula() can generate syntactically invalid formulas
   File: oracle/symbolic/modifiable.py:238-246
   Issue: Random formula generation can produce expressions like
   `(fibonacci fibonacci) Fibonacci)` with mismatched parentheses.
   Directive: Implement proper formula validation after generation.
   Use ast.parse() to verify the generated formula is valid Python
   before returning it. Regenerate if invalid (max 3 attempts).

PERFORMANCE AND RESOURCE MANAGEMENT
================================================================================

14. HIGH: Repeated sqlite3.connect() calls create connection overhead
    Files: oracle/memory/archive.py (every method), oracle/memory/chromosome_archive.py
    Issue: Each method call opens a new SQLite connection. This is O(n)
    connection overhead per operation.
    Directive: Implement a connection pool or use a single persistent
    connection with proper thread safety. Use contextmanager pattern:
    ```python
    @contextmanager
    def _get_connection(self):
        conn = sqlite3.connect(self.db_path)
        conn.execute("PRAGMA journal_mode=WAL")
        try:
            yield conn
        finally:
            conn.close()
    ```

15. MEDIUM: No connection cleanup on exceptions in SQLAlchemy backend
    File: oracle/storage/sqlalchemy_backend.py:99-100
    Issue: `_session()` creates sessions that may not be closed on exception.
    Directive: Use `contextmanager` for all session operations:
    ```python
    @contextmanager
    def _session(self):
        session = self.SessionFactory()
        try:
            yield session
        except Exception:
            session.rollback()
            raise
        finally:
            session.close()
    ```

16. MEDIUM: EvolutionaryEngine evolves entire population synchronously
    File: oracle/evolution/deap_engine.py:84-194
    Issue: The main evolve() loop evaluates all chromosomes sequentially.
    With 50+ chromosomes and complex symbolic systems, this is extremely slow.
    Directive: Integrate ParallelEvaluator (already exists at
    oracle/evolution/parallel_evaluator.py) into the main evolve loop.
    Replace the sequential evaluation loop with parallel evaluation.

17. MEDIUM: Memory leak in GenerationalMemory.successful_patterns
    File: oracle/memory/generational_memory.py:21
    Issue: `successful_patterns: list[dict] = []` grows unboundedly.
    Directive: Add a max size (e.g., 1000) and implement a FIFO eviction
    policy: `if len(self.successful_patterns) > MAX_PATTERNS:
    self.successful_patterns = self.successful_patterns[-MAX_PATTERNS:]`
